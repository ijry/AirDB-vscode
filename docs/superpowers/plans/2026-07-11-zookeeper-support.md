# ZooKeeper Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add ZooKeeper as a NoSQL backend with pure JavaScript protocol access, tree browsing, and read-only znode data preview.

**Architecture:** ZooKeeper is stored under `CacheKey.NOSQL_CONNECTION`, routed through a dedicated `ZooKeeperConnection`, and displayed through dedicated `src/model/zookeeper/*` tree nodes. It does not use SQL dialects, table nodes, or the query workspace.

**Tech Stack:** TypeScript, VS Code extension APIs, Vue 2 connection page, `node-zookeeper-client`, mocked Node tests with `ts-node/register/transpile-only`.

## Global Constraints

- Use `node-zookeeper-client` dependency range `^1.1.3`.
- Do not use the native `zookeeper` npm package.
- Default host is `127.0.0.1`.
- Default port is `2181`.
- Default root path is `/`.
- Default `connectTimeout` is `5000`.
- Default `requestTimeout` is `10000`.
- Store ZooKeeper connections under `CacheKey.NOSQL_CONNECTION`.
- Do not implement znode create/update/delete, ACL editing, watch subscriptions, transactions, query workspace integration, or live integration tests in this version.
- Preserve unrelated untracked `logs/`.

## File Structure

- Modify `package.json`: add `node-zookeeper-client`.
- Modify `package-lock.json`: generated by `npm install`.
- Modify `src/common/constants.ts`: add `DatabaseType.ZOOKEEPER` and ZooKeeper `ModelType` values.
- Modify `src/model/interface/node.ts`: prevent SQL dialect initialization for ZooKeeper and cache ZooKeeper connection nodes by connection id.
- Modify `src/provider/treeDataProvider.ts`: route ZooKeeper to NoSQL storage and instantiate `ZooKeeperConnectionNode`.
- Modify `src/service/connectionManager.ts`: route `DatabaseType.ZOOKEEPER` to `ZooKeeperConnection`.
- Create `src/service/connect/zookeeperConnection.ts`: normalize config, create client, list children, read data/stat, and adapt lifecycle.
- Create `src/model/zookeeper/zookeeperBaseNode.ts`: shared connection lookup and error helpers.
- Create `src/model/zookeeper/zookeeperConnectionNode.ts`: root saved connection node.
- Create `src/model/zookeeper/zookeeperZnodeNode.ts`: recursive znode node and read-only data preview.
- Modify `src/vue/connect/index.vue`: add ZooKeeper logo, database selector entry, defaults watcher, and SSH/SSL visibility behavior.
- Create `resources/icon/zookeeper.svg`: connection page and tree icon.
- Modify `package.json`: add `airdb.zookeeper.znode.open` command and menu entries for opening/copying znodes.
- Create focused ZooKeeper tests under `test/`.
